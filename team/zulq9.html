PROJECT: Inventory Manager

Overview

Inventory manager is for Small to Medium Enterprises (SMEs) who prefer to use a desktop app to manage their shop inventory. More importantly, it is optimized for those who prefer to work with a Command Line Interface (CLI) while still having the benefits of a Graphical User Interface (GUI). It will allow various users to perform multiple inventory management tasks such as creating sales, generating reports, creating purchase orders and more. The user interacts with it using a CLI, and it has a GUI created with JavaFX. It is written in Java, and has about 10 kLoC.

Listing all purchase order : list-po

The list-po command shows a list of all the purchase order.
This command is available to members only.

Figure 4.10. All purchase orders in Inventory Manager listed on screen.

Format: list-po

Adding a purchase order : add-po

The add-po command adds a purchase order for an item based on it’s sku.
The default status of the added purchase order is 'PENDING' This command is available to members only.

Format: add-po s/SKU_NUMBER q/QUANTITY d/REQUIRED_DATE sp/SUPPLIER

Example:

  • add-po s/apple-iphone-xr q/1000 d/2018-12-12 sp/Apple Inc

  • add-po s/samsung-s9 q/12 d/2101-12-12 sp/Samsung

Approving a purchase order : approve-po

The approve-po command approves the specified purchase order from the purchase order list.
Upon approving, the quantity stated in the purchase order will be added to the designated item.
This command is available to the manager only.

Format: approve-po INDEX

For the approve-po command, do take note of the following points:

  • Approves the purchase order at the specified INDEX.

  • The index refers to the index number shown in the displayed inventory list.

  • The index must be a positive integer 1, 2, 3, …​

  • The command can only be applied to purchase order with 'PENDING' status

Examples:

  • list-po
    approve-po 1
    Approves the first purchase order in the purchase order list.

Figure 4.11. Approves the specified purchase order by the it’s index in the list.

Editing a purchase order : edit-po

The edit-po command edits an existing purchase order in the Inventory Manager.
This command is available to members only.

Format: edit-po INDEX [q/QUANTITY] [d/REQUIRED_DATE] [sp/SUPPLIER]

For the edit-po command, do take note of the following points:

  • Update the purchase order at the specified INDEX.

  • The index refers to the index number shown in the displayed purchase order list.

  • The index must be a positive integer 1, 2, 3, …​

  • At least one of the optional fields must be provided.

  • Existing values will be updated to the input values.

  • SKU of the item in the purchase order list is uneditable

  • The command can only be applied to purchase order with 'PENDING' status

Examples:

  • edit-po 2 q/100
    Updates the quantity of the 3rd purchase order to 100.

  • edit-po 1 q/999 d/2018-12-12
    Updates the quantity and required date of the 1st purchase order to 999 and 2018-12-12 respectively.

Deleting a purchase order : delete-po

The delete-po command deletes the specified purchase order from the purchase order list.
This command is available to members only.

Format: delete-po INDEX

For the delete-po command, do take note of the following points:

  • Deletes the purchase order at the specified INDEX.

  • The index refers to the index number shown in the displayed purchase order list.

  • The index must be a positive integer 1, 2, 3, …​

Examples:

  • list-po
    delete-po 5
    Deletes the 5th purchase order in the inventory.
